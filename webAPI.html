Supported dataset types


1) Market Basket list. No header is expected, The number of columns is undefined (Default). If header row exists then in 1st parameter of the extra parameters field the string representing the absent item must be declared. If absent item is nothing or not exists then assign "" or "nan". The participant columns must be declared in extra parameters field starting from the 2nd parameter.

The supported types for excluding redundant rules from the association rule set are listed below.

  1. Interchange LHS and RHS case: The comparable rules are the ones that have reversed their LHS and RHS (e.g. A->B & B->A). The rule with the smaller confidence value is removed.

  2. Redundant Rules with Fixed RHS Removes redundant rules that have multiple items in LHS but have the same RHS itemset. If n number of (n-1) LHS-itemset with the same RHS rules are found in the retrieved rule set, the rule is removed otherwise remains. Example: If XY->Z, X->Z, Y->Z exist in the rule set then XY->Z is removed

  3. Redundant Rules with Fixed LHS Removes redundant rules that have multiple items in RHS but have the same LHS itemset. If n number of (n-1) RHS-itemset with the same LHS rules are found in the retrieved rule set, the rule is removed otherwise remains. Example: If X->YZ, X->Y, X->Z exist in the rule set then X->YZ is removed

The types are selected using the bitwise AND (&) operator. For example, if it is desirable to select the first case then select the value 1 (001); If it is desirable the third one then select 4 (100); If it is desirable the second and third type then select 6 (110). With 0 (000) it does not remove the redundant rules.
